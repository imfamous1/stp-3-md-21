# Задание №1: Создание сайта-визитки на HTML/CSS

**Автор:** Евгений Кокорин  
**Дата:** October 08, 2025

## Цель

Разработать персональный сайт-визитку/портфолио для демонстрации навыков верстки и самопрезентации.

## Технические требования

- Чистый HTML/CSS (без JavaScript, фреймворков, библиотек)

### Обязательное использование:

- **CSS Grid** для основной структуры
- **Flexbox** для компонентов
- **Адаптивная верстка** (мобильные устройства + десктоп)
- **Семантическая разметка HTML5**

## Содержание сайта

### Шапка (Header)

- Имя, фамилия
- Должность/специальность
- Навигационное меню

### Главный раздел (Hero section)

- Фотография/аватар
- Краткое представление (2-3 предложения)
- Кнопка "Связаться"

### О себе (About)

- Биография
- Навыки (hard/soft skills)
- Интересы

### Опыт работы (Experience)

- Реальный опыт (если есть)
- Или блок "Опыта работы нет" + акцент на обучении

### Образование (Education)

- Учебные заведения
- Курсы, сертификаты

### Контакты (Contacts)

- Email
- Телефон
- Соцсети (LinkedIn, GitHub и т.д.)
- Форма обратной связи (простая)

### Футер (Footer)

- Копирайт
- Повтор основных контактов

## Процесс работы

### 1. Дизайн и планирование

Изучите примеры на Tilda: https://tilda.cc/ru/madeontilda/ (раздел "Портфолио")

Используйте языковые модели для помощи с дизайном:

- Делайте скриншоты понравившихся сайтов
- Детально описывайте: цвета, шрифты, расположение элементов
- Просите предложить цветовые палитры и типографику

### 2. Разработка

- Начните с мобильной версии (mobile-first)
- Протестируйте на разных устройствах
- Оптимизируйте изображения
- Проверьте семантику и доступность

### 3. Публикация

- Создайте репозиторий на GitHub
- Загрузите файлы проекта
- Проверьте пути к изображениям (относительные пути)
- Активируйте GitHub Pages в настройках репозитория
- Протестируйте опубликованный сайт

## Критерии оценки

- ✓ Корректность верстки (Grid/Flex)
- ✓ Адаптивность
- ✓ Качество дизайна и UX
- ✓ Полнота информации
- ✓ Работоспособность на GitHub Pages
