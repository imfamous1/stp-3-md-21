# Практическое задание 2: Интерактивная панель управления UI-компонентами

## Описание задания

Создайте веб-приложение — тематический дашборд, которое демонстрирует создание, управление и взаимодействие различных UI-компонентов (виджетов) с использованием принципов ООП.

### Требования к тематике

Тема дашборда должна быть осмысленной и практически применимой. Примеры:

- Дашборд аналитики
- Персональный организатор
- Мониторинг системы
- Игровая статистика
- И другие темы на ваше усмотрение

### Обязательные требования

- Из всех виджетов должно быть **как минимум два**, которые получают данные со сторонних API (материал про API в следующей лекции)
- Весь проект должен быть выгружен на **GitHub** с активным **GitHub Pages**

## Архитектура проекта

Примерные требования к архитектуре и реализации (могут быть изменены на ваше усмотрение):

### Структура файлов

Проект должен быть разбит на модули (ES6 Modules):

```
index.html          - основной файл, подключающий main.js как модуль (type="module")
main.js             - главный файл приложения, отвечающий за инициализацию и общую логику
js/                 - директория для модулей с классами
├── UIComponent.js  - базовый (абстрактный) класс для всех виджетов
├── ToDoWidget.js   - класс виджета "Список дел"
├── QuoteWidget.js  - класс виджета "Случайная цитата"
└── Dashboard.js    - класс, управляющий всей панелью (коллекция виджетов)
styles/             - директория для стилей
├── main.css        - основные стили приложения
└── components.css - стили для виджетов
```

## Требования к классам

### Базовый класс UIComponent

Должен иметь:

- **Конструктор**, принимающий конфигурацию (например, `title`, `id`)
- **Метод `render()`**, который возвращает DOM-элемент виджета (обертку)
- **Метод `destroy()`**, который корректно удаляет виджет из DOM и очищает слушатели событий
- **Общие методы** для всех виджетов (например, `minimize`, `close`)

### Конкретные классы виджетов (наследуются от UIComponent)

#### ToDoWidget

Реализует функционал из старого задания:

- Добавление задач
- Удаление задач
- Отметка выполненных задач

**Требования:**

- Вся логика должна быть инкапсулирована внутри класса
- Данные списка дел должны храниться как свойство этого класса

#### QuoteWidget

Отображает случайную цитату и кнопку "Обновить".

**Требования:**

- При нажатии цитата меняется
- Цитаты можно хранить в массиве прямо в классе

### Класс Dashboard

**Обязательные методы:**

- `addWidget(widgetType)` — добавляет виджет указанного типа
- `removeWidget(widgetId)` — удаляет виджет по ID

**Функциональность:**

- Инкапсулирует в себе массив (коллекцию) активных виджетов
- Отвечает за отрисовку общей сетки или списка виджетов

### Главный файл main.js

Должен:

- Создавать экземпляр `Dashboard`
- Реализовывать логику кнопок "Добавить виджет ToDo", "Добавить виджет Quote" и т.д., которые вызывают методы дашборда
- Управлять общей структурой страницы

## Функциональные требования

1. **Динамическое добавление и удаление виджетов** разных типов
2. **Независимое состояние** каждого виджета (список задач, текущая цитата)
3. **Корректное удаление слушателей событий** при удалении виджета (используйте `removeEventListener` или делегирование событий)
4. **Чистый, читаемый CSS** (рекомендуется использовать CSS Grid или Flexbox для layout панели)
