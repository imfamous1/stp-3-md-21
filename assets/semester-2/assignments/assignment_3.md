# Практическое задание 3: Игра-платформер на JavaScript

## Описание задания

Создайте веб-игру в жанре платформер (platformer) с использованием JavaScript. Игра должна быть полностью функциональной, включать управление персонажем, физику движения, коллизии и игровую механику.

### Требования к игре

Игра должна содержать основные элементы платформера:

- Управляемый персонаж (прыжки, движение влево/вправо)
- Платформы и препятствия
- Система коллизий (столкновения)
- Система подсчета очков или прогресса
- Условия победы/поражения (опционально)
- Несколько уровней или бесконечный режим (на выбор)

## Технические требования

### Выбор технологий

**Основной стек:**
- HTML5 Canvas API (рекомендуется) или DOM-манипуляции
- CSS для стилизации интерфейса
- Vanilla JavaScript (чистый JavaScript)

**Допустимые библиотеки и инструменты:**

- Canvas API для отрисовки графики
- Игровые библиотеки (Phaser.js, Matter.js для физики — опционально)
- Аудио библиотеки для звуковых эффектов (опционально)
- Любые другие библиотеки по согласованию

**Важно:** Основная логика игры должна быть реализована на чистом JavaScript.

### Обязательное требование: использование нейросетей

Задание **обязательно** должно быть выполнено с использованием нейросетей. Допустимые инструменты:

- DeepSeek
- ChatGPT
- Perplexity
- Cursor
- Другие аналогичные инструменты

**Важно:** В игре или в файле README необходимо указать:
- Какую нейросеть вы использовали
- Сколько времени заняло выполнение задания
- Какие части игры были созданы с помощью нейросети

### Требования к функционалу

#### Обязательные элементы:

1. **Управление персонажем:**
   - Движение влево/вправо (клавиши A/D или стрелки)
   - Прыжки (клавиша Space или W)
   - Плавная анимация движения

2. **Игровая физика:**
   - Гравитация
   - Коллизии с платформами
   - Обработка столкновений

3. **Игровой интерфейс:**
   - Отображение счета/очков
   - Индикатор здоровья (опционально)
   - Кнопки управления (старт, пауза, рестарт)

4. **Игровая логика:**
   - Система подсчета очков
   - Условия завершения уровня/игры
   - Обработка игровых событий

#### Дополнительные возможности (для повышения оценки):

- Несколько уровней с разной сложностью
- Сбор предметов (монеты, бонусы)
- Враги или препятствия
- Система сохранения прогресса (localStorage)
- Звуковые эффекты и музыка
- Анимации персонажа и объектов

## Архитектура проекта

### Рекомендуемая структура файлов:

```
index.html          - основной файл с canvas или игровым контейнером
js/
├── main.js         - главный файл, инициализация игры
├── Game.js         - основной класс игры (управление игровым циклом)
├── Player.js       - класс персонажа (движение, физика)
├── Platform.js     - класс платформы
├── Collision.js    - система обработки коллизий
├── InputHandler.js - обработка пользовательского ввода
└── Renderer.js     - класс для отрисовки (опционально)
css/
└── style.css       - стили для интерфейса игры
assets/             - ресурсы (изображения, звуки - опционально)
```
