# Выпускные проекты по дисциплине «Современные технологии разработки»

## 1. Технические требования к проекту

Проекты выполняются с использованием **HTML, CSS и JavaScript**. Допускается применение фреймворков (React, Vue, Angular и др.) при наличии соответствующих компетенций у студента.

## 2. Объём проекта

Количество страниц определяется логикой проекта и не регламентируется жёстко. Допустимы как многостраничные, так и одностраничные (SPA) решения.

## 3. Повторение тем в группе

Одна тема может быть выбрана **не более чем двумя студентами**.

## 4. Срок защиты

**Зачётная неделя / экзаменационная сессия.**

## 4.1. Размещение проекта

**Важно:** Итоговый проект должен быть выложен на GitHub и GitHub Pages (или любой другой хостинг). На защите итогового проекта у преподавателя должен быть доступ к онлайн-версии и к репозиторию.

## 4.2. Требования к выполнению

**Критически важно:** Выполнение всех пунктов, которые заявлены в теме проекта. Без выполнения всех обязательных пунктов работа будет считаться невыполненной. Это также относится к корректной работе всех элементов интерфейса — если функционал не реализован, его лучше не отображать.

## 5. Критерии оценивания

### Дизайн и пользовательский опыт (UX/UI)

- Современный и адаптивный дизайн
- Удобство навигации и логическая структура
- Соответствие принципам визуальной коммуникации
- **Важно:** Не принимается стандартный UX/UI от нейросетей. Шаблоны, сгенерированные нейросетями, хорошо прослеживаются и должны быть изменены под собственный макет. Рекомендуется сначала поработать над уникальным макетом проекта.

### Функциональность и интерактивность

- Корректная работа всех элементов интерфейса. **Важно:** Если какой-то функционал не работает, его лучше не отображать, так как это будет считаться недоделанной работой.
- Реализация заявленной JavaScript-логики
- Обработка пользовательских действий и данных

### Содержательное наполнение

- Полнота раскрытия темы
- Качество контента и его организация
- Семантическая вёрстка и доступность

### Техническое качество

- Чистота и структурированность кода
- Валидность вёрстки и соответствие стандартам

### Архитектура и масштабируемость

- Логическое разделение кода на модули
- Возможность расширения функционала
- Использование современных подходов к разработке

## Темы проектов

### 1. Умное портфолио с фильтрацией

**Задача:** Реализовать динамическую фильтрацию проектов по технологиям и сложности с сохранением состояния в LocalStorage. В портфолио должно быть представлено **не менее 8 проектов** для фильтрации.

### 2. E-commerce с корзиной

**Задача:** Создать систему корзины с расчётом стоимости, применением промокодов и сохранением данных. Поскольку у нас нет бэкенда, необходимо подготовить JSON-файл для отправки данных на сервер. При оформлении заказа должен формироваться JSON-файл с данными заказа, который будет передаваться на бэкенд.

### 3. Блог с CRUD

**Задача:** Реализовать создание, редактирование и удаление статей с валидацией форм и превью контента.

### 4. Новостной портал с поиском

**Задача:** Настроить поиск по статьям с фильтрацией по дате и категориям, используя Debounce. Новости необходимо получать через API, желательно российские источники.

### 5. Ресторан с онлайн-заказом

**Задача:** Реализовать приложение для заказа, когда пользователь пришёл в ресторан и за столиком делает заказ. При оформлении заказа пользователю должно отображаться предварительное время приготовления. **Очень важно учитывать каждый момент:** если заказано 10 одинаковых блюд, время заказа также увеличивается. Данные должны быть актуальными и логичными (например, не должно быть ситуаций, когда лимонад наливается 10 минут). Время приготовления должно рассчитываться на основе реальной логики работы ресторана, а не придуманных нейросетью данных.

### 6. Event-платформа с регистрацией

**Задача:** Создать систему регистрации на события с валидацией данных и отправкой подтверждения. Данные о регистрациях должны отправляться в табличном виде в Google Docs или другую CRM-систему (при наличии соответствующих компетенций у студента). Речь идёт об администрировании этой задачи — со стороны разработчика должна быть возможность видеть, сколько людей регистрируется на события.

### 7. Интерактивная фотогалерея с управлением состоянием

**Задача:** Реализовать систему тегов и категорий для фильтрации изображений с сохранением выбранных фильтров в URL. Должна быть реализована кнопка "Поделиться ссылкой с фильтрами", позволяющая отправить другу ссылку с применёнными фильтрами.

### 8. Образовательная платформа с тестами

**Задача:** Настроить систему прохождения тестов с таймером, подсчётом баллов и отображением прогресса. На платформе должно быть представлено **не менее 4 тестов**, каждый из которых содержит **минимум 20 вопросов**. У каждого теста должна быть разная степень сложности и одна предметная область (например, тест по JavaScript: начинающий, средний, продвинутый, эксперт).

### 9. Фитнес-трекер с графиками

**Задача:** Реализовать ввод тренировок, построение графиков прогресса и расчёт статистики. Необходимо внедрить профиль пользователя с заполнением его данных: пользователь указывает возраст, рост, вес. На основе этих данных при указании информации о тренировке должно примерно подсчитываться количество потраченных калорий. Также должна быть добавлена возможность отслеживания питания, где пользователь указывает блюда и примерно подсчитывается количество калорий. Можно использовать основные блюда для примера.

### 10. Сервис заказа услуг с календарём

**Задача:** Создать систему бронирования для любой предметной области (парикмахер, маникюр, кино и т.д.). Основная идея — календарь, который отображает доступное время. При бронировании какого-то времени пользователю не должно предоставляться оно как доступное. В качестве базы данных можно использовать JSON-файл, в котором учитывается доступное время для бронирования.

## Дополнительные возможности выбора темы проекта

### 11. Реализация части дипломного проекта

Студенты могут выбрать для веб-реализации один из модулей своего дипломного проекта, согласовав объем и функциональность с преподавателем.

### 12. Индивидуальная тема по согласованию

При наличии собственной идеи проекта, студент может предложить персональную тему через личный кабинет преподавателя с обоснованием её практической ценности и технической реализации.
