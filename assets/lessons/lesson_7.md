# Блоковая модель (Box Model). Основа верстки

Каждый элемент в HTML - прямоугольный блок. CSS Box Model - набор правил, который определяет, как вычисляются размеры этого блока (его ширина и высота) и как он взаимодействует с соседними элементами.

## Модель

### 1. Контент

Содержимое, область где находится текст, изображение и т.д. Управляется свойствами `width` (ширина) и `height` (высота).

### 2. Padding

Внутренний отступ, прозрачная область вокруг содержимого, внутри границы `border`. Создает "воздух" между контентом и рамкой. Задается свойством `padding`, или `padding-left`, `padding-right`, `padding-top`, `padding-bottom`.

### 3. Border

Граница, линия или узор, которая окружает `padding` и контент. Задается свойством `border`, внутри свойства `border` есть `width`, `style`, `color`.

Примеры:
- `border: 1px solid black;`
- `border-width: 1px;`
- `border-style: solid;`
- `border-color: black;`

### 4. Margin

Внешний отступ, прозрачная область вне границы, она отделяет текущий элемент от соседей. Задается свойством `margin`, `margin-top`, `margin-left`, `margin-right`, `margin-bottom`.

## Проблема "сломанной" математики

По умолчанию стандартная блочная модель: свойства `width` и `height` задают размеры только для области `content`.

Мы задали `width: 200px`, а по факту получили блок шириной 250px (если добавили padding и border).

### Решение: `box-sizing: border-box`

Это свойство меняет логику расчета и говорит браузеру, что свойства `width` и `height` должны включать в себя `padding` и `border`.

```css
* {
    box-sizing: border-box;
}
```

Все современные фреймворки и подходы к верстке используют `border-box`. Рекомендуется даже применять его ко всем элементам на странице с помощью универсального селектора `*`.
